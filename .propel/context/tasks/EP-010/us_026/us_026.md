# User Story - US_026

## Story ID

- ID Format: US_026

## Story Title

- Medical Code Suggestion & Review

## Description

- As a healthcare provider, I want AI-powered suggestions for ICD-10 and CPT medical codes based on clinical documentation and encounter notes, with a review and approval workflow, so that medical coding is accurate, faster, and reduces claim denials.

## Acceptance Criteria

### AC-1: AI generates code suggestions from clinical notes

- **Given**: A provider completes an encounter and clinical notes are saved
- **When**: The coding suggestion job processes the encounter
- **Then**: Gemini AI analyzes the notes and suggests relevant ICD-10 diagnosis codes and CPT procedure codes with confidence scores (0-1) and rationale for each suggestion, within 30 seconds of note completion

### AC-2: Code suggestions displayed with context

- **Given**: AI code suggestions are generated for an encounter
- **When**: The provider or coder views the coding review screen
- **Then**: Each suggested code is displayed with: code number, description, confidence score, the specific text excerpt supporting the suggestion (highlighted), and alternative codes ranked by relevance

### AC-3: Provider review and approval workflow

- **Given**: A provider views the suggested codes for their encounter
- **When**: They review the suggestions
- **Then**: They can: accept a code (added to encounter), reject a code (logged with reason), modify the code (select alternative from searchable code database), or add manual codes not suggested by AI

### AC-4: Code validation against payer rules

- **Given**: Codes are selected (AI-suggested or manually entered) for an encounter
- **When**: The provider finalizes the coding
- **Then**: The system validates: code combinations are valid per CMS rules, no duplicate codes exist, required modifiers are present, and diagnosis codes support procedure codes, displaying warnings for any violations

### AC-5: Coding accuracy feedback loop

- **Given**: A provider accepts or rejects AI-suggested codes over time
- **When**: The feedback is accumulated (minimum 50 coding decisions)
- **Then**: The system tracks AI accuracy metrics per provider/specialty, identifies patterns in rejections, and the prompt to Gemini is refined based on common correction patterns (human-in-the-loop improvement)

## Edge Cases

- Clinical notes too brief for confident coding → return top suggestions with confidence < 0.5 and prominently display "Insufficient documentation for confident coding — additional notes recommended"
- Rare condition with no clear ICD-10 match → suggest closest parent code with qualifier; flag for specialist coder review
- Coding suggestions generated but encounter later amended → re-run code suggestion on amended notes; show diff of previous vs new suggestions
