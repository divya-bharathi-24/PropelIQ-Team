# User Story - US_021

## Story ID

- ID Format: US_021

## Story Title

- Clinical Document Upload

## Description

- As a patient, I want to upload clinical documents (lab results, imaging, referrals, prescriptions) securely with metadata tagging, so that my healthcare providers have access to my complete medical documentation for informed clinical decisions.

## Acceptance Criteria

### AC-1: Secure document upload with file validation

- **Given**: An authenticated patient navigates to the document upload section
- **When**: They select files to upload
- **Then**: The system accepts PDF, JPEG, PNG, and TIFF formats up to 10MB per file and 50MB total per session, validates file integrity (MIME type verification, not just extension), and rejects invalid files with specific error messages

### AC-2: Metadata tagging during upload

- **Given**: A patient uploads a document
- **When**: The upload form is presented
- **Then**: Required metadata fields include document type (lab result, imaging, referral, prescription, other), date of document, provider who issued it (optional), and free-text description; system auto-suggests category based on filename

### AC-3: Upload progress and confirmation

- **Given**: A patient submits a document for upload
- **When**: The upload is in progress
- **Then**: A progress bar shows percentage complete, upload can be cancelled mid-progress, and upon completion a confirmation shows the document thumbnail, metadata summary, and processing status

### AC-4: Document stored with encryption at rest

- **Given**: A document upload completes
- **When**: The file is persisted
- **Then**: It is stored with AES-256 encryption at rest, the file reference (not the file itself) is recorded in the database, and an immutable audit record logs who uploaded what and when

## Edge Cases

- Upload interrupted by network failure → resume capability for files >2MB using chunked upload; completed chunks are retained for 24 hours
- Patient uploads same document twice → detect duplicate by file hash (SHA-256); warn "This document appears to already be uploaded" with option to proceed or cancel
- Malicious file disguised as PDF → server-side MIME type validation (magic bytes), virus scan before processing, quarantine if suspicious
