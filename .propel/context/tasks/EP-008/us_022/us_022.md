# User Story - US_022

## Story ID

- ID Format: US_022

## Story Title

- Document Processing Pipeline

## Description

- As a healthcare provider, I want uploaded clinical documents automatically processed with OCR text extraction and AI-powered summarization, so that I can quickly review key information without reading entire documents.

## Acceptance Criteria

### AC-1: OCR text extraction on upload completion

- **Given**: A document (PDF/image) is successfully uploaded
- **When**: The processing pipeline triggers (Hangfire background job)
- **Then**: OCR extracts all text content within 60 seconds for documents up to 10 pages, stores the extracted text in the OcrResults table with page-level segmentation, and updates document status to "text_extracted"

### AC-2: AI-powered document summarization

- **Given**: OCR has successfully extracted text from a document
- **When**: The summarization step runs
- **Then**: Gemini AI (gemini-1.5-flash) generates a structured summary containing: key findings, dates mentioned, medications listed, provider recommendations, and follow-up actions, stored as JSON with the document

### AC-3: Extracted data linked to patient record

- **Given**: A document summary is generated
- **When**: The pipeline completes
- **Then**: Extracted medications are cross-referenced with the patient's current medication list, identified conditions are suggested for addition to medical history, and all extractions are flagged as "AI-suggested" requiring provider verification

### AC-4: Processing status visible to patient and provider

- **Given**: A document is in the processing pipeline
- **When**: Either the patient or provider views the document
- **Then**: A status indicator shows the current stage (Uploaded → Processing → Text Extracted → Summarized → Complete), with estimated time remaining based on queue depth

### AC-5: Gemini rate limit management for batch processing

- **Given**: Multiple documents are queued for AI summarization
- **When**: The processing rate approaches 15 RPM limit
- **Then**: The pipeline throttles requests to 12 RPM (leaving headroom for real-time requests), queues overflow documents, and processes them in FIFO order without losing any documents

## Edge Cases

- OCR fails on poor quality scan → mark as "manual review required", notify patient "Document quality too low for automatic processing — staff will review manually"
- Document in unsupported language → detect language; if not English, mark for manual review with detected language tag
- Gemini returns low-confidence summary (confidence < 0.6) → flag summary as "low confidence", do not auto-suggest data linkages, present to provider with prominent warning
