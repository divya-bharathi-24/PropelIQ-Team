# User Story - US_020

## Story ID

- ID Format: US_020

## Story Title

- Multi-Channel Reminders

## Description

- As a patient, I want to receive appointment reminders via my preferred channels (email, SMS, push notification) at configurable intervals, so that I am reliably notified about upcoming appointments and can confirm or cancel in advance.

## Acceptance Criteria

### AC-1: Reminder schedule with configurable intervals

- **Given**: A confirmed appointment exists
- **When**: The reminder engine evaluates the appointment
- **Then**: Reminders are scheduled at 72 hours, 24 hours, and 2 hours before the appointment (default intervals), with each reminder sent via the patient's preferred channel

### AC-2: Multi-channel delivery (email, SMS, push)

- **Given**: A reminder is due
- **When**: The Hangfire job fires
- **Then**: The reminder is sent via the patient's preferred channel (configurable in profile settings); if email, sent via SendGrid; if SMS, sent via configured SMS provider; if push, sent via Firebase Cloud Messaging

### AC-3: Reminder contains actionable response options

- **Given**: A patient receives a reminder
- **When**: They read the notification
- **Then**: It includes appointment details (date, time, provider) and actionable buttons/links: "Confirm", "Reschedule", and "Cancel" that perform the respective action with one click

### AC-4: Confirmation response tracked

- **Given**: A patient clicks "Confirm" in a reminder
- **When**: The confirmation is processed
- **Then**: The appointment status is updated to "patient_confirmed", the no-show risk score is re-evaluated (lowered), and no further confirmation reminders are sent (information reminders still proceed at 2 hours)

### AC-5: Escalated reminders for high-risk appointments

- **Given**: An appointment is flagged as high no-show risk (score > 60)
- **When**: The standard reminder schedule applies
- **Then**: Additional reminders are sent at 48 hours and 6 hours before, and if no confirmation is received by 12 hours before, a staff follow-up task is auto-created

## Edge Cases

- Patient has no email/phone → attempt push notification only; if push not registered, create staff task to call patient manually
- SendGrid daily limit reached (100/day free tier) → queue remaining emails; prioritize by appointment proximity (sooner = higher priority)
- Reminder sent but appointment already cancelled by provider → check appointment status immediately before sending; if cancelled, send "appointment cancelled" notification instead
