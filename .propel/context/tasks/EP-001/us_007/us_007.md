# User Story - US_007

## Story ID

- ID Format: US_007

## Story Title

- Patient Self-Registration

## Description

- As a patient, I want to register for an account with email verification and profile setup, so that I can securely access the platform and manage my healthcare appointments.

## Acceptance Criteria

### AC-1: Registration form captures required patient data

- **Given**: A new patient navigates to the registration page
- **When**: They fill in the registration form
- **Then**: The form collects first name, last name, email, phone, date of birth, and password with real-time validation (email format, password strength 8+ chars with uppercase/number/special)

### AC-2: Email verification sent on registration

- **Given**: A patient submits a valid registration form
- **When**: The account is created
- **Then**: A verification email is sent via SendGrid within 30 seconds containing a time-limited token (24-hour expiry), and the account status is set to "pending_verification"

### AC-3: Account activation on email confirmation

- **Given**: A patient receives the verification email
- **When**: They click the verification link within 24 hours
- **Then**: The account status changes to "active", the patient role is assigned, and they are redirected to the login page with a success message

### AC-4: Duplicate email prevention

- **Given**: An email address already exists in the system
- **When**: A new patient attempts to register with the same email
- **Then**: The system returns a generic "registration failed" message (not revealing whether the email exists) and does not create a duplicate account

### AC-5: Password hashing with bcrypt

- **Given**: A patient submits their registration with a password
- **When**: The account is persisted
- **Then**: The password is hashed using bcrypt with cost factor 12 before storage, and the plaintext password is never logged or stored

## Edge Cases

- Verification email not received → resend button available after 60 seconds cooldown; max 3 resends per hour per email address
- Registration attempt with disposable email domains → allow (do not block) but flag for manual review
- Browser closed during registration → partial data not persisted; user must restart registration
