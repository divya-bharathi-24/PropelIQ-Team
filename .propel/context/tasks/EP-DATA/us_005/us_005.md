# User Story - US_005

## Story ID

- ID Format: US_005

## Story Title

- Clinical & Audit Database Schemas

## Description

- As a development team member, I want PostgreSQL schemas designed and migrated for clinical domains (Document, ClinicalIntelligence, Notification) and the immutable audit log, so that HIPAA-compliant data storage is available for clinical features and compliance tracking.

## Acceptance Criteria

### AC-1: Document domain schema created

- **Given**: PostgreSQL 16 is provisioned with the document schema
- **When**: EF Core migrations complete
- **Then**: Tables for Documents, DocumentVersions, DocumentMetadata, and OcrResults are created with columns for file references (not inline BLOBs), MIME types, and processing status enums

### AC-2: ClinicalIntelligence domain schema created

- **Given**: The clinical_intelligence schema exists
- **When**: Migrations complete
- **Then**: Tables for AiSuggestions, MedicalCodes, ConflictDetections, RiskScores, and IntakeResponses are created with JSON columns for AI model outputs and indexes on (patient_id, created_at)

### AC-3: Notification domain schema created

- **Given**: The notification schema exists
- **When**: Migrations complete
- **Then**: Tables for NotificationTemplates, NotificationLogs, and DeliveryAttempts are created with indexes on (patient_id, channel, status) and TTL-based archival column for logs older than 90 days

### AC-4: Immutable audit log schema created

- **Given**: The audit schema exists
- **When**: Migrations complete
- **Then**: An append-only AuditEvents table is created with columns (event_id, actor_id, action, resource_type, resource_id, timestamp, payload_json), no UPDATE/DELETE permissions granted, and a partitioned index on (timestamp, resource_type)

### AC-5: AES-256 encryption at rest for PII columns

- **Given**: Tables containing PII (patient name, SSN, medical records) exist
- **When**: Data is written to these columns
- **Then**: EF Core value converters encrypt data using AES-256 before storage and decrypt on read, with encryption keys managed via environment variable injection

## Edge Cases

- Audit table grows beyond free-tier storage (1GB) → partition by month with automated archival of partitions older than 12 months to cold storage
- Encryption key rotation → support dual-key reading (old + new) during rotation window; background job re-encrypts with new key
- OcrResults contain extremely large JSON → enforce 500KB max payload with overflow stored as file reference
