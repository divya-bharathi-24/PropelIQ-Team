# User Story - US_004

## Story ID

- ID Format: US_004

## Story Title

- Core Domain Database Schemas

## Description

- As a development team member, I want PostgreSQL schemas designed and migrated for core domains (Auth, Patient, Appointment, Provider), so that feature teams have a normalized, performant data layer to build upon.

## Acceptance Criteria

### AC-1: Schema-per-service isolation implemented

- **Given**: PostgreSQL 16 is provisioned
- **When**: EF Core migrations run for Auth, Patient, Appointment, and Provider contexts
- **Then**: Each service has its own schema (auth, patient, appointment, provider) with no cross-schema foreign keys, enforcing bounded context isolation

### AC-2: Auth domain tables created

- **Given**: The auth schema exists
- **When**: Migrations complete
- **Then**: Tables for Users, Roles, UserRoles, RefreshTokens, and LoginAttempts are created with appropriate indexes on email (unique), role lookups, and token expiry

### AC-3: Patient domain tables created

- **Given**: The patient schema exists
- **When**: Migrations complete
- **Then**: Tables for Patients, PatientContacts, InsuranceDetails, and MedicalHistory are created with PII columns marked for encryption and composite indexes on (last_name, date_of_birth)

### AC-4: Appointment domain tables created

- **Given**: The appointment schema exists
- **When**: Migrations complete
- **Then**: Tables for Appointments, TimeSlots, ProviderSchedules, and SlotSwapRequests are created with indexes on (provider_id, date), (patient_id, status), and constraint ensuring no overlapping slots per provider

### AC-5: EF Core 8 configuration with Npgsql

- **Given**: All domain schemas are defined
- **When**: Application startup completes
- **Then**: DbContext per service resolves correctly, connection pooling is configured (min 2, max 10 for free tier), and migrations can be applied idempotently via CLI

## Edge Cases

- Migration conflict on parallel development → migration naming convention includes timestamp prefix; CI rejects duplicate migration names
- Connection pool exhaustion under load → circuit breaker returns 503 after 5 seconds timeout rather than queuing indefinitely
- Schema drift between environments → CI pipeline compares pending migrations and blocks deployment if unapplied migrations exist
