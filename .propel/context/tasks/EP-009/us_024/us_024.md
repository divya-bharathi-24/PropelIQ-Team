# User Story - US_024

## Story ID

- ID Format: US_024

## Story Title

- Data Conflict Detection & Resolution

## Description

- As a healthcare provider, I want the system to automatically detect conflicting data across patient records (duplicate entries, contradictory medications, inconsistent demographics) and provide AI-assisted resolution suggestions, so that patient data integrity is maintained without manual cross-referencing.

## Acceptance Criteria

### AC-1: Automated conflict detection on data ingestion

- **Given**: New data is entered or imported for a patient (from intake, document processing, or manual entry)
- **When**: The data is persisted
- **Then**: A conflict detection job compares new data against existing records, identifying: duplicate entries (fuzzy match >85%), contradictory values (e.g., different DOB across records), and logically inconsistent data (e.g., medication + known allergy)

### AC-2: AI-powered resolution suggestions

- **Given**: A conflict is detected
- **When**: Gemini AI analyzes the conflicting data points
- **Then**: It generates a resolution suggestion with confidence score, rationale (which source is likely authoritative based on recency, source type, and data quality signals), and recommended action (merge, keep newer, flag for manual review)

### AC-3: Conflict dashboard for providers

- **Given**: Unresolved conflicts exist for patients in a provider's panel
- **When**: The provider views the conflict dashboard
- **Then**: Conflicts are listed sorted by severity (Critical: medication/allergy conflicts first, then High: demographic mismatches, then Medium: duplicate entries), with patient name, conflict type, and age of conflict

### AC-4: One-click resolution with audit trail

- **Given**: A provider reviews a conflict with an AI resolution suggestion
- **When**: They accept the suggestion or choose a manual resolution
- **Then**: The resolution is applied atomically, the conflicting record is archived (not deleted), the resolution is logged in the audit trail with provider ID and chosen action, and the conflict is removed from the dashboard

### AC-5: Conflict prevention on entry

- **Given**: A staff member is entering data that would create a conflict
- **When**: They attempt to save
- **Then**: A warning is displayed immediately showing the existing conflicting data and suggesting correction before save, without blocking the save if they explicitly override

## Edge Cases

- Same conflict detected by multiple concurrent processes → deduplicate using conflict signature hash; only one conflict record persists
- AI resolution confidence below 0.5 → do not auto-suggest; mark as "Requires Manual Review" with both data points displayed side-by-side
- Conflict involves legally protected data (e.g., reproductive health in certain states) → restrict conflict visibility to authorized roles per data sensitivity classification
