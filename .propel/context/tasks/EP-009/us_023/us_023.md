# User Story - US_023

## Story ID

- ID Format: US_023

## Story Title

- 360-Degree Patient View Aggregation

## Description

- As a healthcare provider, I want a unified 360-degree patient view that aggregates data from all platform services (appointments, intake, documents, medications, risks, billing) with AI-powered health insights, so that I can make informed clinical decisions with a complete patient picture in a single screen.

## Acceptance Criteria

### AC-1: Unified patient dashboard aggregates cross-service data

- **Given**: A provider selects a patient
- **When**: The 360-degree view loads
- **Then**: Data is fetched from all relevant services (Appointment, Intake, Document, ClinicalIntelligence, Notification) and rendered within 3 seconds, showing: demographics, upcoming/past appointments, intake summaries, document list, medication list, allergy alerts, and risk scores

### AC-2: AI-generated health timeline

- **Given**: A patient has historical data across multiple visits
- **When**: The provider views the timeline tab
- **Then**: Gemini AI generates a chronological health narrative highlighting significant events, treatment changes, and trend patterns, presented as an interactive timeline with expandable detail nodes

### AC-3: Clinical alerts and contraindication warnings

- **Given**: A patient's aggregated data contains potential issues
- **When**: The 360-degree view loads
- **Then**: AI-identified alerts are displayed prominently (top of page, red/orange severity coloring): drug interactions, allergy conflicts with planned treatments, overdue screenings, and abnormal lab trends

### AC-4: Real-time data freshness indicators

- **Given**: The 360-degree view is displaying patient data
- **When**: Data is sourced from cache or stale sources
- **Then**: Each data section shows a "Last updated" timestamp and a freshness indicator (green=< 5 min, yellow=5-60 min, red=> 60 min or unavailable), with a manual refresh button per section

### AC-5: Data section graceful degradation

- **Given**: One or more backend services are unavailable
- **When**: The 360-degree view loads
- **Then**: Available sections render normally, unavailable sections show "Data temporarily unavailable" placeholder with retry button, and the page does not fail entirely due to a single service outage

## Edge Cases

- Patient has no data in a section (new patient) → show "No data yet" with contextual CTA (e.g., "Schedule first appointment", "Upload documents")
- AI health timeline generation exceeds rate limit → show "Generating timeline..." placeholder; deliver asynchronously when available; cache generated timeline for 24 hours
- Provider views patient while another provider is updating records → display "Record being edited by Dr. X" banner; show last committed data; refresh on edit completion via WebSocket
