# User Story - US_025

## Story ID

- ID Format: US_025

## Story Title

- Staff Verification Workflow

## Description

- As a staff member, I want a structured workflow to verify AI-aggregated and AI-suggested patient data before it becomes part of the official medical record, so that clinical data accuracy is maintained with human oversight on all AI-generated content.

## Acceptance Criteria

### AC-1: Verification queue for AI-generated data

- **Given**: AI has generated suggestions (document summaries, extracted data, health insights)
- **When**: A staff member opens the verification queue
- **Then**: All pending verifications are listed with patient name, data type (medication, allergy, condition, summary), AI confidence score, and time since generation, sorted by confidence (lowest first for highest uncertainty)

### AC-2: Side-by-side source and suggestion view

- **Given**: A staff member selects a verification item
- **When**: The detail view opens
- **Then**: The original source (document image/text, intake response) is displayed alongside the AI extraction/suggestion, with differences highlighted and confidence scores per field

### AC-3: Approve, reject, or modify workflow

- **Given**: A staff member reviews an AI suggestion
- **When**: They take action
- **Then**: They can: Approve (data becomes official record), Reject (data is discarded with reason), or Modify (edit the AI suggestion before approving), and all actions are recorded with staff ID and timestamp

### AC-4: Verification SLA tracking

- **Given**: AI-generated data has been pending verification
- **When**: It exceeds the 24-hour SLA
- **Then**: The item is escalated (highlighted in red, moved to top of queue), a notification is sent to the team lead, and if it exceeds 48 hours, it is auto-assigned to the next available staff member

## Edge Cases

- Staff verifies data then patient uploads contradicting document → re-trigger conflict detection; create new verification item referencing both the approved data and new document
- Bulk verification needed (100+ items after batch document processing) → provide "batch approve" for items with confidence > 0.95 from the same source type; each still gets individual audit record
- Staff modifies AI suggestion to something that creates a NEW conflict → run conflict detection on the modified data before persisting; warn staff if new conflict would be introduced
