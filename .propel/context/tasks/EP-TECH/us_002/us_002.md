# User Story - US_002

## Story ID

- ID Format: US_002

## Story Title

- .NET 8 Backend & API Gateway Scaffolding

## Description

- As a development team member, I want the .NET 8 backend scaffolded with 7 microservice projects, YARP API Gateway, REST/JWT/OpenAPI configuration, so that all backend feature development can proceed on a consistent, secure API infrastructure.

## Acceptance Criteria

### AC-1: .NET 8 solution with 7 microservice projects created

- **Given**: No backend solution exists in the repository
- **When**: The backend scaffolding is executed
- **Then**: A .NET 8 solution is created containing 7 minimal API projects (Auth, Appointment, Intake, Document, ClinicalIntelligence, Notification, Admin) each with Controller/Service/Repository layers and dependency injection configured

### AC-2: YARP API Gateway configured as single entry point

- **Given**: The 7 microservice projects exist
- **When**: The API Gateway project is configured
- **Then**: YARP reverse proxy routes all client requests to downstream services, validates JWT Bearer tokens on every request, and returns 401 for unauthenticated calls

### AC-3: REST API with OpenAPI 3.0 documentation

- **Given**: Each microservice has at least one endpoint
- **When**: The application starts
- **Then**: Swashbuckle generates an OpenAPI 3.0 specification accessible at `/swagger`, documenting all endpoints with request/response schemas

### AC-4: TLS 1.2+ enforced on all endpoints

- **Given**: The API Gateway is running
- **When**: A client attempts an HTTP (non-TLS) connection
- **Then**: The connection is rejected or redirected to HTTPS, and all inter-service communication uses TLS 1.2 or higher

### AC-5: JWT Bearer authentication configured

- **Given**: The Auth microservice is running
- **When**: A valid JWT token is presented in the Authorization header
- **Then**: The request is authenticated, claims are extracted, and role-based access is enforced at the gateway level

## Edge Cases

- Microservice fails to start → YARP health check marks it unhealthy and returns 503 for that service's routes within 60 seconds
- JWT token expired mid-request → return 401 with `token_expired` error code, not a generic 403
- OpenAPI spec generation fails silently → CI pipeline validates swagger.json exists and is valid JSON on every build
