# User Story - US_001

## Story ID

- ID Format: US_001

## Story Title

- Angular 17 Frontend Project Scaffolding

## Description

- As a development team member, I want the Angular 17 frontend project scaffolded with Standalone Components, route-based lazy loading, Angular Material, and PWA support, so that all feature epics can build UI components on a consistent, optimized frontend foundation.

## Acceptance Criteria

### AC-1: Angular 17 project initialized with Standalone Components

- **Given**: No frontend project exists in the repository
- **When**: The scaffolding script is executed
- **Then**: An Angular 17 project is created with `standalone: true` as the default for all generated components, no NgModules present, and TypeScript strict mode enabled

### AC-2: Route-based lazy loading configured for role dashboards

- **Given**: The Angular 17 project is initialized
- **When**: The routing module is configured
- **Then**: Three lazy-loaded route groups exist (Patient, Staff, Admin), each producing a separate chunk, and the initial bundle size is under 300KB gzipped

### AC-3: Angular Material integrated with WCAG 2.1 AA theme

- **Given**: The Angular 17 project has routing configured
- **When**: The UI library is installed and configured
- **Then**: Angular Material 17.x is installed, a custom theme is applied, and all Material components meet WCAG 2.1 AA color contrast requirements

### AC-4: PWA manifest and service worker registered

- **Given**: The Angular 17 project is built for production
- **When**: The application is served
- **Then**: A valid PWA manifest is present with app name, icons, and theme color; the service worker caches static assets and previously visited pages for offline access

### AC-5: Reactive Forms module available globally

- **Given**: The Angular 17 project is initialized
- **When**: A new component needs form handling
- **Then**: ReactiveFormsModule is importable in any standalone component without additional configuration, and RxJS operators are available for reactive state management

## Edge Cases

- Initial bundle exceeds 300KB gzipped due to Angular Material → tree-shake unused components and verify with `ng build --stats-json`
- Service worker conflicts with dev environment hot reload → disable service worker in development mode via environment flag
- Multiple developers scaffolding simultaneously → lock the project init to a single CI pipeline step
