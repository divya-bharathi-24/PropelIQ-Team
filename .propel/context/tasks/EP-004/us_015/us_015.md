# User Story - US_015

## Story ID

- ID Format: US_015

## Story Title

- Calendar Sync

## Description

- As a patient, I want to sync my appointments with external calendar applications (Google Calendar, Outlook, Apple Calendar), so that my healthcare appointments appear alongside my personal schedule without manual entry.

## Acceptance Criteria

### AC-1: ICS feed URL generated per patient

- **Given**: An authenticated patient navigates to calendar sync settings
- **When**: They enable calendar sync
- **Then**: A unique, unguessable ICS feed URL is generated that external calendar apps can subscribe to, containing all future confirmed appointments

### AC-2: Google Calendar integration via OAuth

- **Given**: A patient selects "Connect Google Calendar"
- **When**: They complete the OAuth 2.0 consent flow
- **Then**: Their healthcare appointments are pushed to a dedicated "Healthcare" calendar in Google Calendar within 2 minutes, with create/update/delete sync

### AC-3: Appointment changes reflected in external calendar

- **Given**: A patient has calendar sync enabled
- **When**: An appointment is booked, rescheduled, or cancelled
- **Then**: The external calendar is updated within 5 minutes, showing correct time, provider, and status; cancelled appointments are removed (not just marked)

### AC-4: Calendar disconnect and data cleanup

- **Given**: A patient wants to stop calendar sync
- **When**: They click "Disconnect" for a connected calendar
- **Then**: The OAuth token is revoked, the ICS feed URL is invalidated, and a confirmation message explains that existing calendar entries created by the app will remain but no longer update

## Edge Cases

- OAuth token expires → attempt silent refresh; if refresh fails, mark sync as "disconnected" and notify patient to reconnect
- Patient has 100+ appointments → ICS feed paginated to include only future appointments plus last 5 completed; full history available on-demand
- External calendar service unavailable → queue sync operations with exponential backoff (max 6 retries over 1 hour); show "sync delayed" status in UI
