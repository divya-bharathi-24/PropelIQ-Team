# User Story - US_010

## Story ID

- ID Format: US_010

## Story Title

- Patient Profile & Dashboard

## Description

- As a patient, I want a comprehensive profile management page and a personalized dashboard showing upcoming appointments, recent activity, and quick actions, so that I can manage my healthcare information and navigate the platform efficiently.

## Acceptance Criteria

### AC-1: Patient profile view and edit

- **Given**: An authenticated patient navigates to their profile
- **When**: The profile page loads
- **Then**: It displays personal information (name, email, phone, DOB, address), insurance details, and emergency contacts with an edit button that enables inline editing with save/cancel actions

### AC-2: Profile photo upload

- **Given**: A patient is on their profile page
- **When**: They upload a profile photo
- **Then**: The image is validated (max 2MB, JPEG/PNG only), resized to 200x200px, stored securely, and displayed as their avatar across the platform within 5 seconds

### AC-3: Dashboard displays upcoming appointments

- **Given**: A patient has future appointments booked
- **When**: They land on the dashboard
- **Then**: The next 5 upcoming appointments are displayed as cards showing provider name, specialty, date/time, and status, sorted by nearest date first

### AC-4: Dashboard quick actions panel

- **Given**: A patient is on the dashboard
- **When**: The page loads
- **Then**: Quick action buttons are displayed for "Book Appointment", "Upload Document", "View Medical History", and "Contact Support" with single-click navigation to the respective features

### AC-5: Recent activity feed

- **Given**: A patient has platform activity (bookings, cancellations, document uploads)
- **When**: They view the dashboard
- **Then**: The 10 most recent activities are displayed in chronological order with activity type icon, description, and relative timestamp (e.g., "2 hours ago")

## Edge Cases

- Patient has no upcoming appointments → display "No upcoming appointments" with prominent "Book Now" CTA
- Profile update with invalid phone format → inline validation error; previous valid value preserved until successful save
- Dashboard data partially unavailable (one service down) → render available sections with "Unable to load" placeholder for failed sections; do not block entire dashboard
